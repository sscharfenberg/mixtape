# Internationalization (i18n)

> Part of MixTape v2 (Phase 2). See [../CLAUDE.md](../CLAUDE.md) for the overview and
> [app-rewrite.md](app-rewrite.md) for the surrounding frontend conventions.

MixTape ships in **German (default) and English**, and any user can switch language from the user menu.
The choice is remembered per user (in the database when logged in, in the session for guests), and a
first-time guest is greeted in their browser's language when it is one we support.

The design is a faithful port of [cantrip.me](https://cantrip.me). The load-bearing idea is a **two
translation stores** split:

- **Client catalog** — everything rendered by Vue (labels, buttons, headings, `aria-label`s). Lives in
  `resources/app/lang/{de,en}.json` and is consumed through **`vue-i18n`** with build-time **type-safe
  keys**.
- **Server catalog** — everything emitted by Laravel (validation errors, flash/toast messages, emails,
  the `app:invite` CLI). Lives in `lang/{de,en}/*.php` and is consumed through `__()` / `trans()`.

No message catalog is ever shipped through Inertia — the server shares only the active locale *string*;
the client loads the matching JSON itself (code-split, one chunk per locale).

## How it is set up

### Server: resolving and persisting the locale

| Piece | File | Role |
| --- | --- | --- |
| Supported set | `app/Enums/Locale.php` | The single source of truth for which languages exist (`De`, `En`). `Locale::values()` → `['de','en']`. |
| Resolution | `app/Http/Middleware/ConfigureLocale.php` | Picks the locale each request and calls `app()->setLocale()`. |
| Registration | `bootstrap/app.php` | `ConfigureLocale` is appended to the `web` group **before** `HandleInertiaRequests`, so the locale is active when Inertia shares it and when `<html lang>` renders. |
| Default / fallback | `config/app.php`, `.env` | `APP_LOCALE=de`, `APP_FALLBACK_LOCALE=de`, `APP_FAKER_LOCALE=de_DE`. |
| Per-user preference | `app/Models/User.php` + migration `database/migrations/2026_07_18_000000_add_locale_to_users_table.php` | `users.locale` — `string(8)`, default `de`, enum-cast, in `#[Fillable]`. `User implements HasLocalePreference` (`preferredLocale()`) so **queued mail/notifications auto-localize** to the recipient's language. |
| Switch endpoint | `app/Http/Controllers/LocaleController.php` + `routes/web.php` | `POST /lang/{locale}` (route name `locale`). Validates against the enum (422 on a bad code), persists to `users.locale` (authenticated) or the session (guest), returns JSON. |
| Sharing to the client | `app/Http/Middleware/HandleInertiaRequests.php` | Shares `locale` (the resolved string) and `supportedLocales` (`Locale::values()`). **Not** the messages. |

**Resolution precedence** (in `ConfigureLocale`): authenticated user's `users.locale` → guest session
`locale` → browser `Accept-Language` (parsed, validated against the enum, and written into the session
so it sticks). Anything unrecognised falls back to `config('app.locale')`.

### Client: vue-i18n

| Piece | File | Role |
| --- | --- | --- |
| Instance | `resources/app/i18n.ts` | A module-singleton wrapper around `createI18n` (composition mode, `legacy: false`). Exposes `getI18n`, `setupI18n`, `setI18nLanguage` (also sets `<html lang>`), and `loadLocaleMessages`. |
| Bootstrap | `resources/app/main.ts` | Reads `locale` / `supportedLocales` from the initial Inertia props, calls `setupI18n`, `app.use(i18n)`, and **defers `app.mount()` until the active locale's catalog has loaded** (no flash of raw keys). |
| Catalogs | `resources/app/lang/{de,en}.json` | One nested JSON per locale. Loaded **lazily**: `import(\`./lang/${locale}.json\`)` → Vite code-splits one chunk per locale, so only the active language is downloaded. |
| Typed keys | `resources/types/i18n.d.ts` | Augments vue-i18n's `DefineLocaleMessage` from `typeof de.json`, so **`t('…')` / `$t('…')` keys are checked by `vue-tsc`** — a typo or missing key fails `npm run build`. **`de.json` is the schema authority.** |
| Shared-prop types | `resources/types/inertia.d.ts` | Declares `locale: string` and `supportedLocales: string[]`. |

Usage in components: `const { t } = useI18n()` then `t('header.userMenu.login')`. Outside component
setup (e.g. the async handlers in `resources/app/composables/useTwoFactorAuth.ts`) reach the singleton
via `(getI18n().global as unknown as Composer).t(key)`.

### Client catalog conventions

- **Namespaces**: `common.*` (shared atoms — close, show/hide password, the required-fields hint, …),
  `header.*`, `footer.*`, `auth.*`, `dashboard.*`.
- **Interpolation**: `t('key', { name })` with `{name}` in the message.
- **Embedded components** (an `<icon>` or `<Link>` inside a sentence): use the global `<i18n-t>`
  component with named slots that match `{placeholder}` tokens, e.g.

  ```vue
  <i18n-t keypath="common.requiredFieldsHint" scope="global">
      <template #icon><icon name="required" /></template>
  </i18n-t>
  ```

- German uses the informal **"du"**.

### The language switcher

`resources/app/components/Landmarks/Header/UserMenu/LanguageSwitch.vue` sits in the user menu next to the
theme switch. It renders one `popover-list-item` per `supportedLocales` entry — a **flag image** plus the
language's **endonym** (its native name, e.g. "Deutsch", "English") — with `--selected` marking the
active one. Picking a language: loads that catalog, flips `i18n.global.locale` (reactive re-render) and
`<html lang>`, then fire-and-forget `POST /lang/{locale}` to persist. No Inertia visit, no reload.

Flags live in `resources/app/assets/flags/<locale>.svg` and are resolved with
`new URL(\`../../../../assets/flags/${loc}.svg\`, import.meta.url)` (Vite rewrites the path and bundles
every SVG in that folder at build). They are sized by the `.flag` component style
(`resources/app/styles/components/_flag.scss`) from the `s.$c-flag` token
(`resources/app/styles/abstracts/sizes/components/_flag.scss`) — a fixed `1.5rem × 1rem` box with
`object-fit: cover`.

### Server catalog layout

`lang/de/` and `lang/en/` are structurally identical:

| File | Contents |
| --- | --- |
| `validation.php` | The full Laravel validation message set + `custom` / `attributes`. |
| `auth.php` | Login failure, wrong password, throttle, unverified/invalid-link, 2FA code errors. |
| `passwords.php` | Password-reset broker messages. |
| `flash.php` | Toast messages flashed by controllers/responses (resolved server-side in the request locale, shipped to the toast via the `flash.message` shared prop). |
| `mail.php` | Notification email subjects / greetings / lines. |
| `invite.php` | Output of the `app:invite` console command (uses `trans_choice`). |
| `rules.php` | Messages for the custom rules (`PasswordEntropy`, `ValidInvite`). |

## Adding a new language

Worked example: adding **French (`fr`)**. Substitute your own code (the bare ISO-639 code we key on,
e.g. `fr`, `es`, `it`).

1. **Register the locale** — add a case to `app/Enums/Locale.php`:

   ```php
   case Fr = 'fr';
   ```

   This is all that is needed for the server to accept it, for `POST /lang/fr` to validate, and for
   `fr` to appear in `supportedLocales` (so the switcher renders it automatically).

2. **Add the flag image** — `resources/app/assets/flags/fr.svg`. Requirements:
   - **Filename = the exact locale code**, lowercase, `.svg` (`fr.svg`) — the switcher resolves the file
     by locale code, so a mismatch means a broken image.
   - **Format: SVG only** (bundled and minified by `vite-plugin-image-optimizer`). Keep it small and
     simple — a handful of `<path>`/`<rect>`s. Add a `<desc>` naming the flag.
   - **Give any internal `id`s a unique prefix** (see `en.svg`'s `uk-s` / `uk-t`) so they can never
     collide with another flag's ids.
   - It is rendered into a fixed **3:2 box** (`object-fit: cover`, rounded corners), so a roughly 3:2
     source looks best; other ratios are cropped, not distorted.
   - No sprite rebuild is needed — flags are plain `<img>` assets, **not** part of the icon sprite
     (`npm run icons`).

3. **Add the endonym** — in `LanguageSwitch.vue`, extend the `endonyms` map with the language's own name
   (not a translation): `fr: "Français"`.

4. **Translate the client catalog** — create `resources/app/lang/fr.json` by copying
   `resources/app/lang/de.json` (the schema authority) and translating **every value**. Keep the key
   structure identical to `de.json`. Any key you omit falls back to the fallback locale (`de`) at
   runtime rather than breaking — but aim for completeness. (`de.json` drives the typed keys; adding
   `fr.json` does not change the type schema.)

5. **Translate the server catalog** — create `lang/fr/` with all seven files translated from `lang/de/`:
   `validation.php`, `auth.php`, `passwords.php`, `flash.php`, `mail.php`, `invite.php`, `rules.php`.
   These back `__()` for validation, flash toasts, emails, and the invite CLI.

6. **(Optional) Faker** — no change required; only relevant if you seed locale-specific fake data.

7. **Verify** — `npm run build` (lint + `vue-tsc` typed-key check + Vite; the new flag and `fr.json`
   chunk should appear in the output) and `php artisan test` (79 tests). The `users.locale` column is
   `string(8)`, so it already fits any realistic locale code.

Nothing else references the language list explicitly — the enum feeds the shared `supportedLocales`, the
switcher iterates that, and the flag + catalogs are resolved by code. So steps 1–5 are the whole job.

### Changing the *default* language

Set `APP_LOCALE` (and usually `APP_FALLBACK_LOCALE`) in `.env` / `.env.example` and the `config/app.php`
defaults. The last-resort literals `"de"` in `i18n.ts` (`setupI18n` default) and `main.ts` (used only
when the server prop is somehow absent) can be updated to match, but the server prop normally wins.

## Where a given string belongs

- **Rendered by Vue** (template text, `aria-label`, a computed label) → **client catalog**
  (`resources/app/lang/*.json`), referenced with a typed `t('…')` key.
- **Emitted by Laravel** (validation, a flashed toast, an email, a CLI line, an exception message) →
  **server catalog** (`lang/<code>/*.php`), referenced with `__('file.key')`.

Flash/toast messages are the bridge: the controller/response flashes a **server-translated** string
(`__('flash.…')`, resolved in the request locale), `HandleInertiaRequests` shares it as `flash.message`,
and `ToastContainer.vue` renders it — so those stay server-side, not in the JSON catalog.
